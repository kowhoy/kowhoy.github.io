了解Spark

Spark是开源的分布式查询和处理引擎，具有MapReduce的灵活性和可扩展性，但是速度更快，当数据存储在内存中，速度比hadoop快100倍，存储在磁盘中快10倍。

  • 任何Spark应用程序都会分离主节点上的单一驱动进程(可以包含多个作业)，然后将执行进程(包含多个任务)分配到多个工作节点。
    任何工作节点都可以执行来自不同作业的多个任务

  • 弹性分布式数据集(RDD)
    弹性分布式数据集是不可变Java虚拟机(JVM)对象的分布式集合，Spark就是围绕RDD构建的。

    1. RDD的计算可以使用到缓存和存在内存的模式进行，速度要比hadoop快一个数量级；
    2. RDD提供一些粗粒度的数据转换(如 map/reduce/filter)，保持了Hadoop的灵活性和扩展性；
    3. RDD以并行方式应用和记录数据转换，提高了速度和容错力；

  1. RDD提供了数据沿袭，保护了RDD避免数据丢失，如果一个RDD分区丢失，会有足够的信息重新创建丢失分区。

    RDD两组并行操作：转换(transformation) 和 动作(action)

    1. transformation: 在已经存在的RDD基础上创建新的RDD，返回指向新RDD的指针，是惰性(lazy)操作，即：不会立即计算结果，只有遇到action，并且需要将结果返回给驱动程序时，才会计算转换；

    2. action: 在执行相关的操作之后返回结果

  • DataFrame
    DataFrame像RDD一样，是分布在集群的节点中的不可变的数据集合，不同的是，DataFrame中的数据是以命名列的方式组织的，与pandas中的DataFrame、关系型数据库表类似
    作用：使大型数据集的处理更加容易；与Java和Scala相比，Python中的RDD非常慢，DataFrame的引入使性能在各语言中保持稳定。
  • Dataset
    DataSet是分布式的数据集合，Dataset提供了强类型支持，也是在RDD的每行数据加了类型约束。DataSet是在Spark1.6中添加的新的接口。它集中了RDD的优点（强类型和可以用强大lambda函数）以及使用了Spark SQL优化的执行引擎。PySpark目前不支持，因为Python不是一种类型安全的语言。

在Spark2.0中，DataFrame和DataSet进行了统一，DataFrame是未类型化的DataSet的一个别名
DataFrame = DataSet[Row]

弹性分布式数据集

一. RDD的内部运行方式

  • RDD是并行操作，Spark工作原理的最大优势是：每个转换并行执行可以大大提高速度。
  • 数据集转换通常为惰性

二. 创建RDD

有两种方式创建RDD，分别为parallelize 和 textFile

  • parallelize 将list或者array转化为RDD
    data = sc.parallelize(['A', 'B', 'C'])
  • textFile 从文件系统读取文件转化为RDD
      data_from_file = sc.textFile('file_path', 3)
    第二个参数表示该数据集被划分的分区个数

RDD是无schema的数据结构，所以tuple、dict、list可以混用

例：

data = sc.parallelize([{"a":1}, (1,2,3,4), [4,3,2]])

从文本文件中读入数据，文件中每一行形成RDD的一个元素

三. 全局作用域和局部作用域

  • Spark可以在两种模式下运行: 本地和集群
  • 在本地运行Spark和使用python没有什么不同，只是语法上的不同，数据和代码在不同的工作者进程之间复制
  • 在集群模式下，提交执行任务时，驱动程序节点会为任务创建DAG，决定工作者节点在运行特定的任务，会有一组变量和方法给工作者节点，如果在运行任务时，执行者改变变量或者覆盖这些方法，不会影响到其他的工作者，会产生不可预知的错误。

四. 转换

数据集： highway.txt

  • .map(…)

      data_from_file = sc.textFile('file_path.txt')

  kind_data = data_from_file.map(lambda row: row.split(',')[2])

  #kind_data.collect()

  kind_data.take(10)

  • .filter(…)

      data_filter = kind_data.filter(lambda row: row != 'KIND' and int(row) == 8301)
  data_filter.take(10)

  • .flatMap(…)
    flatMap返回一个扁平的结果，可以将每一行看作一个列表，将所有的记录简单加入到一起

      kind_data_add_str = kind_data.map(lambda row: (row, row+'abc'))
  kind_data_add_str.take(3)
  #[('KIND', 'KINDabc'), ('8301', '8301abc'), ('8301', '8301abc')]

  kind_data_add_str_2 = kind_data.flatMap(lambda row: (row, row+'abc'))
  kind_data_add_str_2.take(3)
  #['KIND', 'KINDabc', '8301']

  kind_data_add_str_3 = kind_data.map(lambda row: row + 'abc')
  kind_data_add_str_3.take(5)
  #['KINDabc', '8301abc', '8301abc', '8302abc', '8302abc']

  kind_data_add_str_4 = kind_data.flatMap(lambda row: row + 'abc')
  kind_data_add_str_4.take(5)
  #['K', 'I', 'N', 'D', 'a']

  • .distinct(…)

      distinct_kind = data_from_file.map(lambda row: row.split(',')[2]).distinct()
  distinct_kind.collect()
  # ['KIND', '8302', '8301']

    ⚠️:这是一个高开销的操作，只有在必要的时候才使用

  • .sample(…)
    sample()方法返回数据集的随机样本，第一个参数指定采样是否应该替换，第二个参数返回数据的分数(分子)， 第三个参数伪随机数产生器的种子

      fraction = 0.1
  sample_data = data_from_file.sample(False, fraction, 666)
  data_sample.take(3)

  • .leftOuterJoin(…), .join(…), .intersection(…)
    相当于SQL中的左连

      rdd1 = sc.parallelize([('a', 1), ('b', 2), ('c', 3)])
  rdd2 = sc.parallelize([('a', 8), ('c', 9), ('d', 10)])

  rdd3 = rdd1.leftOuterJoin(rdd2)
  rdd3.collect()
  # [('a', (1, 8)), ('b', (2, None)), ('c', (3, 9))]

  rdd4 = rdd1.join(rdd2)
  # [('a', (1, 8)), ('c', (3, 9))]

  rdd5 = rdd1.intersection(rdd2) ## 返回两个RDD中相等的记录
  # []

    ⚠️: 这是一个高开销的操作，谨慎使用

  • .repartition(…)
    重新对数据集进行分区，改变数据集分区的数量，此操作会对性能产生巨大的影响

五. 操作

  • .take(…)
    返回数据集，take()方法优于collect()方法，因为take()可以指定数据分区的前n行，collect()返回的是全部RDD，使用.takeSample(...)可以随机记录

      data_from_file.takeSample(False, 3, 666)

  • .collect(…)

  • .reduce(…), reductByKey(…)

      rdd1_num = rdd1.map(lambda row: row[1])
  rdd1_num_sum = rdd1.map(lambda row: row[1]).reduce(lambda x, y: x+y)
  rdd1_num_sum
  # 6

  data_ley = sc.parallelize([('a', 4), ('b', 3), ('c', 2), ('a', 8), ('d', 2), ('b', 1), ['d', 3]], 3)

  data_ley.reduceByKey(lambda x, y: x+y).collect()
  # [('d', 5), ('b', 4), ('a', 12), ('c', 2)]

    ⚠️: reducer传递的函数需要是关联的，即元素顺序改变，结果不变；函数也是可交换的，操作符改变，结果不变

  • .count(…), .countByKey(…)

  • .saveAsTextFile(…)
  • .foreach(…)
    对每一条记录应用一个定义好的函数。当希望数据保存到PySpark本身不支持的数据时，就可以使用，比如print()
    def f(x):
        print(x)

    data_key.foreach(f)

DataFrame

DataFrame是一种不可变的犯不是数据集，被组织成指定的列，类似于关系型数据库的表；引入DataFrame之前，Python查询速度普遍是使用RDD的Scala查询慢两倍，主要消耗在Python和JVM之间的通信开销

创建DataFrame

通常使用SparkSession来导入数据创建DataFrame

在Spark中直接生成DataFrame数据

  • 生成JSON文件

    >>> stringJSONRDD = sc.parallelize(("""
... {"id":"123","name":"Katie", "age":19, "eyeColor":"brown"}""",
... """{"id":"234", "name":"Michael", "age":22, "eyeColor":"green"}""",
... """{"id":"345", "name":"Simone", "age":23, "eyeColor":"blue"}"""))
    • 创建DataFrame
      >>> swimmersJSON = spark.read.json(stringJSONRDD)
>>> swimmersJSON
DataFrame[age: bigint, eyeColor: string, id: string, name: string]

简单的DataFrame查询

  1. .show() 默认显示前十行
>>> swimmersJSON.show()
+---+--------+---+-------+
|age|eyeColor| id|   name|
+---+--------+---+-------+
| 19|   brown|123|  Katie|
| 22|   green|234|Michael|
| 23|    blue|345| Simone|
+---+--------+---+-------+
  1. SQL查询 ## 需要线生成表
    swimmersJSON.createOrReplaceTempView("swimmers")

    spark.sql("select count(*) from swimmers").show()

  • 编程指定模式

>>> stringCSVRDD = sc.parallelize([(123, 'Katie', 19, 'brown'), (234, 'Michael', 22, 'green'), (345, 'Simone', 23, 'blue')])
>>> 
>>> schema = StructType([StructField("id", LongType(), True), StructField("name", StringType(), True), StructField("age", LongType(), True), StructField("eyeColor", StringType(), True)])
>>> 
>>> swimmers = spark.createDataFrame(stringCSVRDD, schema)
>>> 
>>> swimmers.show()
