hadoop生态安装-hadoop

Words count 3.4k Reading time 3 mins.

  • hadoop
  • 大数据

本次的hadoop生态是安装部署在阿里云的centos上的

在服务器上创建一个新用户

ssh onestarko@101.111.111.11

adduser onestarko
passwd onestarko #设置密码

用户添加sudo权限

#1，切换到root用户
su
#2. 查看sudoers位置, 正常应该是 /etc/suders
whereis sudoers
#3. 添加编辑权限
chmod -v u+w /etc/sudoers
#4. vim /etc/sudoers 在root ALL=(ALL) ALL 下增加一行
username ALL=(ALL) ALL
#5. 保存，恢复文件权限
chmod -v u-w /etc/sudoers

在用户根目录下创建文件夹

cd ~
mkdir software 
mkdir app
mkdir data
mkdir source

下载hadoop压缩包

使用是hadoop-2.6.0-cdh5.7.0版本

cd software

wget wget http://archive.cloudera.com/cdh5/cdh/5/hadoop-2.6.0-cdh5.7.0.tar.gz

安装java环境

安装的是1.7版本，由于官网下载需要有登录验证的过程，所以拿下载连接比较困难，因此，现将安装文件下载到本地，然后再上传到阿里云上。

登录https://www.oracle.com/technetwork/java/javase/downloads/index.html找下载地址， 在页面最下方，点击Java Archive的download按钮，找到相应版本下载，这里下载的是jdk-7u51-linux-x64.tar.gz。将本地安装文件上传阿里云:

scp jdk-7u51-linux-x64.tar.gz onestarko@101.111.11.11:/home/onestarko/software

在服务器上进行jdk的安装配置

cd ~/software
tar -zxvf jdk-7u51-linux-x64.tar.gz -C ../app

cd ../app/jdk1.7.0_51/bin
./java -version #验证是否解压正常，有输出版本正常

##配置环境变量
vim ~/.bash_profile
##START
## 在~/.bash_profile增加两行
JAVA_HOME=/home/onestarko/app/jdk1.7.0_51
PATH=$PATH:$JAVA_HOME/bin
## 保存退出
##END

source ~/.bash_profile

java -version #有输出版本正常

机器配置

ssh免密设置

ssh-keygen -t rsa
#一路回车

cp ~/.ssh/id_rsa.pub ~/.ssh/anthorized_keys

ssh 机器实例名 or localhost 是否免密

配置hadoop

# 1.解压
cd ~/software
tar -zxvf hadoop-2.6.0-cdh5.7.0.tar.gz -C ../app/

cd ~/app/hadoop-2.6.0-cdh5.7.0/etc/hadoop

# 2.修改 hadoop-env.sh 中的JAVA_HOME
vim hadoop-env.sh
export JAVA_HOME=/home/onestarko/app/jdk1.7.0_51

# 3.配置core-site.xml
mkdir ~/app/tmp  # 创建tmp目录，因为默认的tmp目录每次系统重启会被删除


vim core-site.xml
<configuration>
        <property>
                <name>fs.defaultFS</name>
                <value>hdfs://onestarko:8020</value>
        </property>

        <property>
                <name>hadooop.tmp.dir</name>
                <value>/home/onestarko/app/tmp</value>
        </property>
</configuration>

# 4.配置 hdfs-site.xml
#文件副本系数
vim hdfs-site.xml
<configuration>
        <property>
                <name>dfs.replication</name>
                <value>1</value>
        </property>
</configuration>

# 5.格式化hdfs(只在第一次执行)
cd ~/app/hadoop-2.6.0-cdh5.7.0/bin
./hdfs namenode -format 

# 6.启动HDFS
cd ~/app/hadoop-2.6.0-cdh5.7.0/sbin
./start-dfs.sh

# 验证是否启动成功
jps

14485 Jps
14113 NameNode
14382 SecondaryNameNode
